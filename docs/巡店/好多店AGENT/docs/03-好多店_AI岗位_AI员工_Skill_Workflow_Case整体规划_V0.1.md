# 好多店 AI Native 业务执行系统

# AI岗位 / AI员工 / Skill / Workflow / Case 整体规划文档 V0.1

## 1. 文档目的

本文定义好多店 AI Native 业务执行系统的业务规划模型。

目标：

不是建设一个聊天机器人，而是在好多店组织中创建 AI 业务岗位，让 AI
像企业员工一样承担明确职责，通过流程、技能和工具完成真实业务闭环。

核心理念：

    AI岗位定义职责
            ↓
    AI员工承担工作
            ↓
    Workflow安排任务
            ↓
    Skill完成能力
            ↓
    Tool访问业务系统
            ↓
    Case形成业务闭环

------------------------------------------------------------------------

# 2. AI岗位体系（AI Role）

## 2.1 定义

AI岗位是业务职责的抽象。

它定义：

-   角色目标
-   工作职责
-   能力范围
-   数据范围
-   权限边界
-   禁止事项

类似企业中的：

-   区域督导
-   店长
-   运营经理

AI岗位不是一个聊天角色，而是一个业务责任角色。

------------------------------------------------------------------------

# 3. AI员工（AI Employee）

## 3.1 定义

AI员工是 AI岗位 的具体实例。

关系：

    AI岗位(Role)

            ↓

    实例化

            ↓

    AI员工(Employee)

例如：

AI风险督导岗位：

可以创建：

-   华东区域AI风险督导
-   华南区域AI风险督导

不同AI员工：

-   负责区域不同
-   门店范围不同
-   权限不同
-   数据范围不同

------------------------------------------------------------------------

# 4. AI员工运行模型

一个AI员工由以下组成：

    AI员工

    +
    岗位定义

    +
    系统提示词

    +
    Harness控制机制

    +
    Skill集合

    +
    Tool权限

    +
    业务数据范围

    +
    Workflow

    +
    Memory

    =
    可执行AI业务角色

------------------------------------------------------------------------

# 5. Skill技能体系

## 5.1 定义

Skill是AI员工完成某类业务任务的方法能力。

一个AI员工不是一个万能模型，而是多个Skill组合。

------------------------------------------------------------------------

# 6. AI风险督导（第一期）

## 6.1 定位

模拟：

区域运营督导。

目标：

接收业务系统产生的风险事件，分析问题，推动整改闭环。

------------------------------------------------------------------------

## 6.2 数据来源

第一阶段：

-   巡店检查结果
-   检查项结果
-   风险规则结果
-   门店信息
-   人员关系
-   整改任务
-   工单记录

暂不接入：

-   视频AI
-   客流
-   财务
-   业绩分析

------------------------------------------------------------------------

## 6.3 Skill设计

### 风险分析Skill

输入：

-   风险事件
-   门店信息
-   检查项
-   历史问题

输出：

-   风险原因
-   证据
-   影响分析

------------------------------------------------------------------------

### 责任定位Skill

输出：

-   店长
-   督导
-   区域负责人

------------------------------------------------------------------------

### 整改方案Skill

输出：

-   整改建议
-   责任人
-   时间要求
-   复查方式

------------------------------------------------------------------------

### Case跟进Skill

负责：

-   判断整改状态
-   判断延期
-   判断是否升级

------------------------------------------------------------------------

# 7. AI店长助手（第二阶段）

## 定位

模拟：

门店店长助手。

目标：

帮助店长提升日常执行效率。

能力：

-   任务提醒
-   巡店问题解释
-   整改指导
-   今日工作安排

Skill：

-   门店日报Skill
-   问题解释Skill
-   工作计划Skill

------------------------------------------------------------------------

# 8. AI运营经理（第二阶段）

## 定位

模拟：

总部运营管理人员。

目标：

辅助区域经营分析和管理决策。

能力：

-   区域分析
-   门店排名
-   风险趋势
-   经营复盘

Skill：

-   经营分析Skill
-   趋势分析Skill
-   运营报告Skill

------------------------------------------------------------------------

    AI Employee

    Step

    ↓

    Skill

    ↓

    Tool

    ↓

    Result

    ↓

    Case

# 11. Case业务闭环

## 11.1 定义

Case代表一个需要持续跟踪的问题。

例如：

    Case:

    杭州西湖店
    冰机卫生整改事件

包含：

-   风险来源
-   AI分析
-   责任人
-   整改任务
-   跟进记录
-   最终结果
