# 好多店 AI Native Skill 体系详细设计文档 V0.1

## 1. 文档定位

本文档定义好多店 AI Native 业务执行系统中的 Skill 体系。

Skill 是 AI 员工完成业务工作的能力单元。

核心思想：

AI员工不是依靠一个万能 Prompt 完成所有任务，而是通过多个专业 Skill
组合完成业务职责。

关系：

    AI岗位(Role)

            |

    AI员工(Employee)

            |

    Workflow

            |

    Skill

            |

    Tool

            |

    业务系统能力

------------------------------------------------------------------------

# 2. Skill定义

## 2.1 什么是Skill

Skill表示：

> AI员工完成某一类业务任务的方法能力。

例如：

AI风险督导：

拥有：

-   风险分析Skill
-   责任定位Skill
-   整改建议Skill
-   Case跟进Skill

------------------------------------------------------------------------

# 3. Skill与Role关系

AI岗位定义：

"需要具备哪些能力"

Skill定义：

"如何完成某项工作"

例如：

    AI风险督导岗位

            |

            |

    Skill集合

            |

    +----------------+

    风险分析Skill

    责任定位Skill

    整改方案Skill

    Case跟进Skill

------------------------------------------------------------------------

# 4. Skill组成模型

一个Skill包含：

    Skill

    ├── 基础信息

    ├── Prompt定义

    ├── 输入Schema

    ├── 输出Schema

    ├── Tool依赖

    ├── 业务规则

    ├── Guard规则

    ├── 示例

    └── 评估指标

------------------------------------------------------------------------

# 5. Skill基础信息

示例：

``` json
{
 "skill_code":"risk_analysis",

 "name":"风险分析",

 "description":"分析门店风险事件原因",

 "version":"1.0",

 "status":"active"
}
```

------------------------------------------------------------------------

# 6. Input Schema设计

Skill必须明确输入。

例如：

风险分析Skill：

输入：

``` json
{
 "enterprise_id":"",

 "store_id":"",

 "risk_alert_id":""
}
```

不允许：

Skill自行猜测业务上下文。

------------------------------------------------------------------------

# 7. Output Schema设计

输出必须结构化。

例如：

``` json
{
 "risk_summary":"",

 "evidence":[
 ],

 "root_cause":"",

 "recommendation":""
}
```

目的：

方便后续Workflow继续处理。

------------------------------------------------------------------------

# 8. Skill与Tool关系

Skill负责：

业务推理。

Tool负责：

访问外部能力。

关系：

    Skill

     |

    调用

     |

    Tool


     |

    查询数据/API/MCP

例如：

风险分析Skill：

需要：

    get_risk_alert_detail

    get_store_profile

    get_patrol_history

------------------------------------------------------------------------

# 9. 第一批Skill规划

# 9.1 AI风险督导Skill

## 风险分析Skill

目标：

分析风险产生原因。

输入：

-   风险事件
-   门店
-   检查项
-   历史记录

输出：

-   风险说明
-   证据
-   原因
-   建议

------------------------------------------------------------------------

## 责任定位Skill

目标：

找到责任关系。

输入：

-   门店
-   区域
-   人员关系

输出：

-   店长
-   督导
-   区域负责人

------------------------------------------------------------------------

## 整改方案Skill

目标：

生成整改建议。

输出：

-   整改措施
-   责任人
-   时间
-   复查方式

------------------------------------------------------------------------

## Case跟进Skill

目标：

持续跟踪问题。

判断：

-   是否完成
-   是否延期
-   是否复发
-   是否升级

------------------------------------------------------------------------

# 13. Skill评估

评估：

## 正确性

是否正确理解业务。

## 数据完整性

是否引用真实业务数据。

## 执行效果

是否推动业务闭环。

## 安全性

是否越权。

------------------------------------------------------------------------

# 14. Skill管理原则

## 原则1

一个Skill解决一类明确业务问题。

不要：

一个Skill包含所有业务。

------------------------------------------------------------------------

## 原则2

Skill必须有输入输出定义。

------------------------------------------------------------------------

## 原则3

Skill不能绕过Tool。

------------------------------------------------------------------------

## 原则4

Skill执行必须可追踪。
