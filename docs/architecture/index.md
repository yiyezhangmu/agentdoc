# 架构设计

记录系统架构、服务治理、权限设计和工程实践的思考与总结。

---

## 📑 知识目录

### 架构模式

- **微服务架构**：服务拆分原则与边界定义
- **事件驱动架构**：异步通信与解耦设计
- **API 网关**：统一入口与流量管理
- **服务网格**：服务间通信与可观测性

### 核心领域

- **权限设计**：RBAC、ABAC 与权限模型
- **数据一致性**：分布式事务与最终一致性
- **容灾设计**：高可用与故障恢复策略
- **性能优化**：缓存、异步处理与资源调度

---

## 🔗 架构决策记录

### 常见架构问题

!!! note "设计原则"

    1. **单一职责**：每个服务只做一件事
    2. **开闭原则**：对扩展开放，对修改关闭
    3. **依赖倒置**：依赖抽象而非具体实现
    4. **接口隔离**：客户端不依赖不需要的接口

## 💡 实践经验

- **服务拆分粒度**：根据业务边界和团队规模确定
- **通信协议选择**：同步用 gRPC/HTTP，异步用消息队列
- **监控体系建设**：日志、指标、链路追踪三位一体
